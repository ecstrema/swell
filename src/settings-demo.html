Settings System Overview

The settings system consists of three main components:

  • Settings Register (settings-register.ts): Manages settings metadata including path, description, type, and options
  • Settings Storage (settings-storage.ts): Handles reading/writing settings to settings.json file (Tauri) or localStorage (web)
  • Settings Page (settings-page.ts): UI component for displaying and editing settings

Example Setting Definition

{
  "path": "Application/Color Theme",
  "description": "Choose the color theme for the application",
  "type": "enum",
  "enumOptions": ["light", "dark", "system"]
}

Supported Setting Types

  • enum: Dropdown selection with predefined options
  • boolean: Checkbox for true/false values
  • string: Text input field
  • number: Number input with optional min/max/step

How to Add New Settings

Register a new setting in settings-register.ts:

settingsRegister.register({
    path: 'Editor/Font Size',
    description: 'Set the editor font size',
    type: 'number',
    defaultValue: 14,
    min: 8,
    max: 32,
    step: 1
});
